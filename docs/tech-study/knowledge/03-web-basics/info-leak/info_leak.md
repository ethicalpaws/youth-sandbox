# 信息泄露
## 什么是信息泄露
1. 信息泄露，是指网站无意中向用户泄露敏感信息根据不同的上下文，网站可能向潜在攻击者泄露各种信息，包括：
其他用户数据：用户名、邮箱、财务信息、身份证号等
敏感商业数据：客户名单、定价策略、商业机密、未发布的产品信息
技术细节：网站架构、使用的框架/库版本、服务器配置、目录结构
2. 为什么技术细节泄露也很危险：
暴露额外的攻击面
提供构建复杂攻击的"拼图缺失块"
帮助攻击者发现其他高危漏洞
## 信息泄露的典型示例
1. 隐藏目录泄露：通过 /robots.txt 或目录列表暴露隐藏目录结构
2. 源代码泄露：临时备份文件（如 .bak、~、.swp）暴露源代码
3. 数据库信息泄露：错误消息中明确提及数据库表名或列名
4. 敏感数据暴露：信用卡号、密码、API密钥在响应中明文显示
5. 硬编码凭证：源代码中硬编码API密钥、IP地址、数据库凭证
6. 行为差异暗示：通过应用行为的细微差异推断资源或用户名的存在/缺失
## 信息泄露的产生原因
1. 未删除内部内容
开发注释、调试代码、测试端点未从生产环境中移除
```
<!-- TODO: 移除这个调试端点，/admin/test-users -->
<div class="hidden">内部API: http://internal-api.company.com/v2/</div>
```
2. 不安全配置
未禁用调试和诊断功能
使用默认配置（默认密码、过于详细的错误消息）
启用了不必要的HTTP方法（如TRACE）
目录列表未禁用
