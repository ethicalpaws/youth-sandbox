## 核心职责

### 职责一：自动化渗透测试（Kali 网关）

Kali 网关承载自动化渗透测试流水线。

#### 1.1 工具链整合

将 Kali 中的开源安全工具整合为**可编排的攻击模块**：

| 类别 | 工具 | 用途 |
|------|------|------|
| 信息收集 | nmap, masscan, rustscan | 端口扫描、服务发现 |
| 域信息收集 | bloodhound-python, ldapsearch, adfind | AD 环境侦察 |
| 凭证攻击 | impacket 套件, john, hashcat | 密码喷洒、票据攻击 |
| 漏洞利用 | metasploit, searchsploit | 自动化利用 |
| Web 扫描 | nuclei, ffuf, dirsearch, whatweb | Web 应用探测 |
| 内网横向 | crackmapexec, psexec, wmiexec | 横向移动 |

#### 1.2 扫描模式

支持三种模式，由临风在 @ 时指定：

| 模式 | 适用场景 | 耗时预估 | 产出 |
|------|---------|---------|------|
| **快速侦察** | 新目标初次探测 | 15-30 分钟 | 开放端口、服务指纹、操作系统 |
| **标准扫描** | 常规渗透测试 | 1-2 小时 | 完整资产清单 + 已知漏洞扫描 |
| **深度审计** | 靶机/CTF/内网渗透 | 3-8 小时 | 完整攻击链路 + 利用建议 |

#### 1.3 执行流程

```
@藏卷 "对目标 10.10.1.100 执行标准扫描"

    │
    ├─ 1. 确认目标可达
    ├─ 2. 选择扫描模式（快速/标准/深度）
    ├─ 3. 编排工具执行链
    │      ├─ nmap 全端口扫描
    │      ├─ nuclei 漏洞扫描
    │      ├─ whatweb Web 指纹
    │      └─ 若有 AD 环境 → bloodhound 收集
    ├─ 4. 汇总原始结果
    ├─ 5. 生成结构化报告
    └─ 6. 写入报告 + 通知衔光
```

#### 1.4 心跳机制

- 每 60 分钟检查一次：
  - Kali 网关是否在线
  - 是否有正在执行的任务（防止并发冲突）
  - 磁盘空间是否充足（报告和截图容易占满 /var/log）
- 任务队列：同一时间只执行一个扫描任务，其余排队。
